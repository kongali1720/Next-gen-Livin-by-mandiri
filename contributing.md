# 🤝 Panduan Kontribusi (Contributing Guidelines)

Terima kasih telah tertarik untuk berkontribusi dalam menyempurnakan gagasan masa depan untuk **Livin' by Mandiri**! Kami sangat terbuka terhadap ide, perbaikan kode backend, optimasi skema API, maupun perbaikan dokumentasi.

## Bagaimana Anda Bisa Berkontribusi?

### 1. Melaporkan Bug atau Usulan Ide
Jika Anda memiliki ide fitur baru atau menemukan kesalahan penulisan dalam dokumen kami, silakan buka **GitHub Issue** baru dengan format:
*   **Judul:** `[USULAN/BUG] - Nama Fitur/Masalah`
*   **Deskripsi:** Penjelasan detail mengenai ide atau kesalahan yang ditemukan serta dampak/manfaatnya bagi aplikasi Livin'.

### 2. Mengirimkan Perubahan (Pull Request)
Jika Anda ingin memperbaiki atau menambahkan contoh kode (seperti skema API OpenAPI baru atau contoh implementasi bahasa pemrograman lain):
1. **Fork** repositori ini.
2. Buat branch baru untuk fitur Anda (`git checkout -b fitur/ide-keren-anda`).
3. Lakukan commit perubahan Anda dengan pesan yang jelas dan deskriptif (`git commit -m 'Menambahkan skema API untuk CBDC'`).
4. Push ke branch Anda (`git push origin fitur/ide-keren-anda`).
5. Buka **Pull Request** ke repositori utama kami dan jelaskan perubahan yang Anda lakukan.

## Pedoman Perilaku
Harap menjaga diskusi tetap profesional, konstruktif, dan fokus pada solusi teknis demi menghasilkan karya tulis terbaik yang dapat menginspirasi industri perbankan nasional.
